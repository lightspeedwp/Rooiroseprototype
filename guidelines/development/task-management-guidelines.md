# Task Management Guidelines

**Version**: 1.0.0  
**Created**: 2026-03-13  
**Purpose**: Standardize task list creation, status tracking, and lifecycle management  
**Applies to**: All files in `/tasks/` folder

---

## 📋 **TASK LIST TYPES**

### 1. Master Task List
**Filename**: `/tasks/task-list.md`  
**Purpose**: Single source of truth for all active and archived tasks  
**Status**: Always active (never archive)  
**Updates**: Every task completion, every cleanup session

### 2. Project Task Lists
**Filename Pattern**: `[project-name]-tasks.md`  
**Example**: `theme-completeness-tasks.md`  
**Purpose**: Detailed task breakdown for specific projects/phases  
**Lifespan**: Active until 100% complete, then archive to `/tasks/archived/YYYY-MM/`

### 3. Current Task Lists
**Filename**: `CURRENT-TASKS.md`  
**Purpose**: Active work-in-progress tasks (current focus)  
**Status**: Always active (overwrites previous CURRENT-TASKS.md when starting new phase)  
**Updates**: Daily or after each task completion

### 4. Orchestrator Task Lists
**Filename Pattern**: `[orchestrator-name]-tasks.md`  
**Example**: `system-stability-audit-tasks.md`  
**Purpose**: Task lists generated by orchestrator prompts  
**Lifespan**: Active until orchestrator complete, then archive

---

## 📂 **TASK LIST NAMING RULES**

### Filename Format
```
[project-name]-tasks.md
```

**Components**:
- `[project-name]` — Lowercase, hyphen-separated (e.g., `theme-alignment`, `memory-optimization`)
- Suffix: Always `-tasks.md`
- Extension: Always `.md` (Markdown)

**Examples**:
- ✅ `theme-completeness-tasks.md`
- ✅ `rooi-rose-content-refactor-tasks.md`
- ✅ `CURRENT-TASKS.md` (special case)
- ❌ `Theme Tasks.md` (spaces)
- ❌ `tasks_list.md` (underscores, wrong suffix)
- ❌ `todo.txt` (wrong extension)

## 🔄 **TASK STATUS MARKERS**

Use these checkbox markers consistently:

| Marker | Meaning | When to Use |
|:-------|:--------|:------------|
| `[ ]` | Todo (Not Started) | Task has not been started yet |
| `[~]` | In Progress | Task is partially complete (some sub-tasks done) |
| `[x]` | Complete | Task is 100% complete (all sub-tasks checked off) |
| `[!]` | Blocked | Task cannot proceed due to external blocker |

**Examples**:
```markdown
- [ ] Task 1.1: Update homepage — Not started
- [~] Task 1.2: Update category pages — 2/5 pages done
- [x] Task 1.3: Update single post — Complete
- [!] Task 1.4: WordPress content import — Blocked (requires admin access)
```

## 📊 **TASK PRIORITY SYSTEM**

### Priority Levels

| Priority | Meaning | Examples | Response Time |
|:---------|:--------|:---------|:--------------|
| **P0** | Critical — Blocks production deployment | Build errors, broken routes, security issues | Immediate |
| **P1** | High — Core features, user-facing improvements | New pages, major features, rebranding | Same day |
| **P2** | Medium — Nice-to-have features, optimizations | Performance improvements, documentation | 1-3 days |
| **P3** | Low — Enhancements, refactoring, cleanup | Code refactoring, minor optimizations | 1-2 weeks |

### How to Assign Priorities

**Ask these questions**:
1. Does this block production deployment? → **P0**
2. Is this a user-facing feature or critical improvement? → **P1**
3. Does this improve performance or DX significantly? → **P2**
4. Is this a nice-to-have or cleanup task? → **P3**

## 🗂️ **TASK LIST LIFECYCLE**

### Creation (Day 0)
1. Create task list in `/tasks/[project]-tasks.md`
2. Add front matter (Created, Status, Priority, Orchestrator)
3. Define all phases and tasks
4. Link from `/tasks/task-list.md` (master list)
5. Reference in `/Guidelines.md` if major project

### Active Period (Days 0 - Completion)
- Task list lives in `/tasks/` (root level)
- Updated after each task completion
- Status % recalculated with each update
- "Last Updated" timestamp refreshed

### Completion (100% Done)
1. Update status to `[x] COMPLETE`
2. Update task overview (100% complete)
3. Create completion report in `/reports/`
4. Link report in task list
5. Update `/Guidelines.md` with completion status
6. Update `/CHANGELOG.md` with changes

### Archive Period (After Completion)
1. Move to `/tasks/archived/YYYY-MM/[filename].md`
2. Update links in `/tasks/task-list.md` to point to archived location
3. Update links in `/Guidelines.md` to archived location
4. Add archive date to task list front matter

### Permanent Record
- Completed task lists remain in archive permanently
- Used for historical reference and project audits
- Never deleted (unlike reports which can be deleted after 90 days)

## ⚠️ **TASK LIST DON'TS**

**DO NOT**:
- ❌ Delete completed task lists (archive them)
- ❌ Use vague task descriptions ("Fix homepage")
- ❌ Skip status markers (always use [ ]/[~]/[x]/[!])
- ❌ Forget to update "Last Updated" timestamp
- ❌ Leave percentages inaccurate (recalculate with each update)
- ❌ Create duplicate task lists for same project
- ❌ Mix task lists with reports (separate files)
- ❌ Forget to update `/tasks/task-list.md` master list

**DO**:
- ✅ Archive completed task lists to `/tasks/archived/YYYY-MM/`
- ✅ Write specific task descriptions with file paths
- ✅ Update status markers after each sub-task
- ✅ Recalculate percentages with each update
- ✅ Mark blocked tasks with [!] and explain blocker
- ✅ Link to related guidelines, reports, orchestrators
- ✅ Provide quick resume prompts for easy continuation
- ✅ Keep `/tasks/task-list.md` synchronized

## 🎯 **QUICK REFERENCE**

**Creating a new task list**:
1. Determine project name (lowercase, hyphen-separated)
2. Use filename: `[project-name]-tasks.md`
3. Copy template from this guideline
4. Fill in all phases and tasks
5. Add to `/tasks/task-list.md` master list
6. Link from Guidelines.md if major project

**Updating task status**:
1. Check off completed sub-tasks `[x]`
2. Update task status marker if all sub-tasks done
3. Recalculate phase completion (X/Y complete)
4. Recalculate total percentage
5. Update "Last Updated" timestamp
6. Update task overview table

**Archiving a completed task list**:
1. Verify 100% complete (all tasks `[x]`)
2. Create `/tasks/archived/YYYY-MM/` if doesn't exist
3. Move task list to archive folder
4. Update links in `/tasks/task-list.md`
5. Update links in Guidelines.md
6. Create completion report in `/reports/`

---

## 💡 **TIPS FOR EFFECTIVE TASK LISTS**

1. **Break down large tasks** — No task should take more than 2 hours
2. **Write clear acceptance criteria** — How do you know it's done?
3. **Estimate time realistically** — Add 30% buffer for unknowns
4. **Update immediately after completion** — Don't let statuses get stale
5. **Use blockers wisely** — Mark tasks that truly cannot proceed
6. **Link related files** — Make it easy to find relevant guidelines/data
7. **Provide resume prompts** — Help AI (and humans) jump back in quickly
8. **Keep master list current** — `/tasks/task-list.md` should always be accurate
