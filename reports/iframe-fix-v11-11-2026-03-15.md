## Problem Statement

User reported `IframeMessageAbortError` continuing to occur despite v11.10 implementation. The error originates from Figma's iframe communication system conflicting with React initialization.

---

## Root Cause Analysis

**v11.10 Limitation**:
- Script was being conditionally injected, but timing issue remained
- Vite could begin loading before iframe detection completed
- React imports could start before abortion logic executed

**Core Issue**:
- Figma's iframe requires complete control over script loading
- ANY React-related script loading triggers the message port error
- Even delayed/conditional loading wasn't aggressive enough

---

## Solution: v11.11 Ultra-Aggressive Protection

### Six-Layer Defense Strategy

**Layer 1: CSS-First Approach**
- Inline `<style>` tag in `<head>` (loads before any JavaScript)
- Class-based show/hide for `#figma-preview` and `#root`
- Instant visual feedback with zero JavaScript dependency

**Layer 2: Immediate Synchronous Detection**
- Blocking `<script>` in `<head>` (executes before any other scripts)
- Detects iframe status synchronously
- Sets THREE global flags for redundancy:
  - `window.__FIGMA_IFRAME_MODE__` (primary)
  - `window.__BLOCK_VITE_SCRIPT__` (script injection blocker)
  - `window.__REACT_DISABLED__` (React initialization blocker)
- Adds `.figma-iframe` class to `<html>` element
- Nullifies `window.React` and `window.ReactDOM`

**Layer 3: Static Preview HTML**
- Always present in DOM
- Shown/hidden via CSS `.figma-iframe` class
- Beautiful branded preview with rooi rose styling

**Layer 4: Conditional Script Injection**
- Checks ALL three flags before creating script tag
- Only injects Vite script if NOT in iframe
- Complete prevention of React loading

**Layer 5: main.tsx Pre-Import Abortion**
- Checks flags BEFORE any imports
- `throw new Error()` to stop execution completely
- Secondary direct iframe detection
- Prevents React/ReactDOM imports

**Layer 6: main.tsx Final Check**
- Checks before `ReactDOM.createRoot()`
- Redundant failsafe if earlier checks bypassed

## Technical Implementation

### Global Flags Set

```javascript
window.__FIGMA_IFRAME_MODE__ = true;   // Primary iframe indicator
window.__BLOCK_VITE_SCRIPT__ = true;   // Prevent script injection
window.__REACT_DISABLED__ = true;      // Disable React init
```

### CSS Class Strategy

```css
/* Default: hide preview, show root */
#figma-preview { display: none; }

/* In iframe: show preview, hide root */
html.figma-iframe #figma-preview { display: flex !important; }
html.figma-iframe #root { display: none !important; }
```

### Script Injection Prevention

```javascript
// Only create script if ALL checks pass
if (!window.__BLOCK_VITE_SCRIPT__ && 
    !window.__FIGMA_IFRAME_MODE__ && 
    !window.__REACT_DISABLED__) {
  // Safe to load React
  var script = document.createElement('script');
  script.src = '/src/main.tsx';
  document.body.appendChild(script);
}
// Otherwise: NO SCRIPT, NO REACT, NO ERROR
```

### Pre-Import Abortion

```typescript
// Throw error BEFORE importing React
if ((window as any).__REACT_DISABLED__) {
  throw new Error('React disabled in Figma iframe mode');
}

// Only import if safe
import React from 'react';
import ReactDOM from 'react-dom/client';
```
