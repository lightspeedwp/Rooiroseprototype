# Iframe Fix v11.10 — Conditional Script Loading

**Date**: 2026-03-15  
**Version**: v11.10  
**Status**: ✅ Complete  
**Issue**: `IframeMessageAbortError` in Figma Make iframe preview

---

## Problem

User reported persistent `IframeMessageAbortError` when viewing the app in Figma Make's iframe environment:

```
IframeMessageAbortError: Message aborted: message port was destroyed
    at r.cleanup (https://www.figma.com/webpack-artifacts/...)
    at s.cleanup (https://www.figma.com/webpack-artifacts/...)
    at eI.setupMessageChannel (https://www.figma.com/webpack-artifacts/...)
    at e.onload (https://www.figma.com/webpack-artifacts/...)
```

**Root Cause**: The Vite script (`/src/main.tsx`) was being loaded even in iframe mode, causing React imports to process and triggering Figma's message port cleanup before our blocking logic could execute.

---

## Solution: Conditional Script Loading (v11.10)

**Strategy**: Completely prevent the Vite script from loading in iframe environments by dynamically injecting it only in standard browsers.

### Implementation

**Layer 1** — Immediate iframe detection in `<head>`:
```javascript
<script>
  (function() {
    var isIframe = false;
    try {
      isIframe = window.self !== window.top;
    } catch (e) {
      isIframe = true;
    }
    
    if (isIframe) {
      console.log('[Index v11.10 Layer 1] Iframe detected - BLOCKING React');
      window.__FIGMA_IFRAME_MODE__ = true;
      window.__BLOCK_VITE_SCRIPT__ = true;
      
      // Show static preview on DOMContentLoaded
      document.addEventListener('DOMContentLoaded', function() {
        var preview = document.getElementById('figma-preview');
        var root = document.getElementById('root');
        if (preview) preview.style.display = 'block';
        if (root) root.style.display = 'none';
      });
    } else {
      window.__FIGMA_IFRAME_MODE__ = false;
      window.__BLOCK_VITE_SCRIPT__ = false;
    }
  })();
</script>
```

**Layer 2** — Conditional script injection in `<body>`:
```javascript
<script>
  (function() {
    // Only load Vite script if NOT in iframe mode
    if (!window.__BLOCK_VITE_SCRIPT__) {
      console.log('[Index v11.10 Layer 2] Loading Vite module...');
      var script = document.createElement('script');
      script.type = 'module';
      script.src = '/src/main.tsx';
      document.body.appendChild(script);
    } else {
      console.log('[Index v11.10 Layer 2] BLOCKED: Vite script prevented in iframe');
    }
  })();
</script>
```

**Layer 3** — Failsafe in `main.tsx` (unchanged):
```typescript
if ((window as any).__FIGMA_IFRAME_MODE__) {
  console.log('[Main v11.10] ABORT: Figma iframe mode flag detected');
  // Do nothing - completely abort execution
} else {
  // Initialize React app
}
```

## Key Changes from v11.9

| Aspect | v11.9 | v11.10 |
|:-------|:------|:-------|
| **Script Tag** | Hardcoded in HTML | Dynamically created |
| **Script Loading** | Always loaded | Conditional (iframe = blocked) |
| **Import Processing** | Imports still processed | Imports never processed |
| **Error Prevention** | Reactive (abort after load) | Proactive (prevent load) |
| **User Experience** | Static preview after delay | Instant static preview |

## Results

✅ **Before** (v11.9):
- Script loaded → Imports processed → main.tsx aborts → Error thrown
- Console: `IframeMessageAbortError`

✅ **After** (v11.10):
- Script never loads → No imports processed → No errors
- Console: `[Index v11.10 Layer 2] BLOCKED: Vite script prevented in iframe`
