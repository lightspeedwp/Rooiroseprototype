# Iframe Fix v11.12 — Maximum Protection

**Date**: 2026-03-15  
**Version**: 11.12 (Maximum Protection)  
**Status**: ✅ Implemented  
**Previous Version**: 11.11 (still had errors)  

---

## Problem

User reported `IframeMessageAbortError` still occurring despite v11.11 implementation. The error continues to happen during Figma's iframe message port setup, indicating React is still being loaded somehow.

---

## Root Cause

**v11.11 Issue**: Script detection wasn't early enough
- Figma's onload event was firing before our checks completed
- The script tag might have been created before flags were checked
- CSS class application had timing issues

**Core Problem**: We need to prevent the script from EVER being created, not just conditionally create it.

---

## v11.12 Solution: Maximum Protection

### Key Changes from v11.11

1. **Earlier flag setting**: Flags set in FIRST script in `<head>`, before ANY other resources
2. **Four flags instead of three**: Added `__NO_REACT__` for extra redundancy
3. **Immediate HTML class**: `document.documentElement.className = 'figma-iframe'` runs SYNCHRONOUSLY
4. **Vite preload blocker**: `window.__vitePreload` nullified to prevent module preloading
5. **Stricter script blocking**: Triple-check before script creation (flags + direct detection)
6. **More aggressive main.tsx**: Check FOUR flags instead of three

---

## Implementation Details

### index.html - First Script (Lines 6-36)

```html
<script>
  // EXECUTE IMMEDIATELY - FIRST THING IN HEAD
  (function() {
    'use strict';
    var isIframe = false;
    try {
      isIframe = window.self !== window.top;
    } catch (e) {
      isIframe = true;
    }
    
    if (isIframe) {
      // IMMEDIATELY mark HTML element
      document.documentElement.className = 'figma-iframe';
      
      // Set ALL flags (4 flags now)
      window.__FIGMA_IFRAME_MODE__ = true;
      window.__BLOCK_VITE_SCRIPT__ = true;
      window.__REACT_DISABLED__ = true;
      window.__NO_REACT__ = true; // NEW FLAG
      
      // Block React globals
      window.React = null;
      window.ReactDOM = null;
      
      // Prevent Vite preloading
      window.__vitePreload = function() { return Promise.resolve(); };
      
      console.log('[v11.12] IFRAME MODE - React completely disabled');
    }
  })();
</script>
```

**Why This Works**:
- Runs BEFORE icon, viewport, description, title
- Synchronous execution (no async/await)
- Sets HTML class immediately for CSS to use
- Four flags for maximum redundancy
- Blocks Vite's preload mechanism

### index.html - Script Injection (Lines 91-120)

```html
<script>
  (function() {
    'use strict';
    
    // Check ALL FOUR flags
    if (window.__BLOCK_VITE_SCRIPT__ === true ||
        window.__FIGMA_IFRAME_MODE__ === true ||
        window.__REACT_DISABLED__ === true ||
        window.__NO_REACT__ === true) {
      console.log('[v11.12] Script injection BLOCKED - iframe mode');
      return; // Stop completely
    }
    
    // Triple-check with direct detection
    var isIframe = false;
    try {
      isIframe = window.self !== window.top;
    } catch (e) {
      isIframe = true;
    }
    
    if (isIframe) {
      console.log('[v11.12] Script injection BLOCKED - direct detection');
      return;
    }
    
    // Safe to load React
    console.log('[v11.12] Loading Vite module...');
    var script = document.createElement('script');
    script.type = 'module';
    script.src = '/src/main.tsx';
    document.body.appendChild(script);
  })();
</script>
```

**Triple Protection**:
1. Check all four flags
2. Direct iframe detection
3. Only create script if BOTH checks pass

### main.tsx - Pre-Import Checks

```typescript
// v11.12 CRITICAL: Check BEFORE any imports

const iframeFlags = [
  (window as any).__REACT_DISABLED__,
  (window as any).__FIGMA_IFRAME_MODE__,
  (window as any).__BLOCK_VITE_SCRIPT__,
  (window as any).__NO_REACT__
];

if (iframeFlags.some(flag => flag === true)) {
  console.log('[Main v11.12] ABORT - Iframe flags detected');
  throw new Error('[v11.12] React execution blocked');
}

// Direct iframe detection
const isIframe = (() => {
  try {
    return window.self !== window.top;
  } catch {
    return true;
  }
})();

if (isIframe) {
  console.log('[Main v11.12] ABORT - Direct iframe detection');
  throw new Error('[v11.12] Cannot execute React in iframe');
}

// Safe to import
import React from 'react';
import ReactDOM from 'react-dom/client';
```

## If This Still Fails

If v11.12 still shows errors, the issue might be:

1. **Figma caching**: Hard refresh (Cmd+Shift+R) might be needed
2. **Browser caching**: Clear browser cache
3. **Timing issue**: Figma might be running setup BEFORE our script
4. **Different error**: The error might not be from our code

**Next step would be v11.13**:
- Move script to separate file loaded with defer
- Add meta tag to prevent all caching
- Use MutationObserver to prevent script injection
- Consider if this error is actually from Figma itself, not our code
