## Violations by Type

### Critical (25 instances) — Typography Inline Styles

**Pattern**: Using `style={{ fontVariationSettings, lineHeight, letterSpacing }}` instead of BEM classes

**Most Common Violators**:
1. **NewsCard.tsx** — 16 instances across 3 card variants
2. **Hero.tsx** — 4 instances (h1, h2 typography)
3. **CompetitionsSection.tsx** — 6 instances (headings)
4. **RelatedContent.tsx** — 4 instances (article/event/edition headings)
5. **Common components** — ErrorBoundary, HeroSlideCard, CompetitionCard, TeamGrid, etc.

**Issue**: These styles are already defined as CSS variables (`--fvs-h1`, `--lh-h2`, `--ls-h3`, etc.) but being applied inline instead of through BEM classes.

**Example Violation**:
```tsx
// ❌ BAD (Inline style)
<h3 
  className="font-normal text-brand-navy has-brand-serif-font-family"
  style={{ 
    fontVariationSettings: "var(--fvs-h3)", 
    lineHeight: 'var(--lh-h3)', 
    letterSpacing: 'var(--ls-h3)' 
  }}
>
  {title}
</h3>

// ✅ GOOD (BEM class)
<h3 className="editorial-heading editorial-heading--h3">
  {title}
</h3>
```

## Appendix: BEM Methodology Reference

### Block
Standalone entity with semantic meaning.
**Example**: `.editorial-heading`, `.article-card`, `.pull-quote`

### Element
Part of a block with no standalone meaning.
**Syntax**: `.block__element`  
**Example**: `.article-card__title`, `.article-card__excerpt`

### Modifier
Flag changing appearance or behavior.
**Syntax**: `.block--modifier`  
**Example**: `.editorial-heading--h1`, `.article-card--featured`
