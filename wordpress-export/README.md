# WordPress Export Directory

**Last updated**: 2026-02-23

This folder contains the complete source code for the "Die Papier" WordPress site migration.

## Directory Structure

*   **`themes/die-papier-theme/`**: The custom Block Theme (FSE).
    *   `theme.json`: V3 design tokens and global styles (V2 palette).
    *   `style.css`: Theme header + backward-compat `:root` variable aliases.
    *   `functions.php`: Theme setup, font enqueues, editor styles.
    *   `patterns/`: 70 block patterns (slug prefix: `die-papier/`).
    *   `templates/`: 27 HTML block templates.
    *   `parts/`: 9 reusable template parts (header, footer, sidebar, etc.).
    *   `styles/`: Global style variations (dark.json, section-red.json, etc.).
    *   `fonts/`: Self-hosted Roboto Serif & Inter webfonts.
*   **`plugins/die-papier-blocks/`**: The custom Functionality Plugin.
    *   `die-papier-blocks.php`: Main plugin file (17 blocks, 7 CPTs, 5 taxonomies).
    *   `src/blocks/`: 17 block source directories (each with block.json, edit.js, index.js, render.php, style.scss, and optionally view.js).
    *   `inc/`: 16 PHP includes (CPTs, taxonomies, REST API, block styles, SCF fields, commerce, JSON-LD, admin columns).
    *   `acf-json/`: ACF/SCF field group exports.
    *   `build/`: Compiled block assets (generated by `npm run build`).
*   **`scripts/`**: Utility scripts for migration.
    *   `generate-wxr.js`: Content export to WordPress WXR format (Node.js).
    *   `seed-commerce.php`: WooCommerce product/subscription setup (WP-CLI).
    *   `validate-wxr.sh`: Automated WXR export + validation (runs generate-wxr.js, checks XML, verifies content counts).
    *   `preflight-check.sh`: Pre-flight deployment check (validates entire directory structure before deployment).
*   **`config/`**: Deploy-ready server configuration files.
    *   `redirects.conf`: Nginx include file (74 redirect rules: 64 static + 10 regex).
    *   `.htaccess-redirects`: Apache mod_rewrite rules (74 redirect rules, mirroring Nginx config).
    *   `redirection-import.json`: WordPress Redirection plugin JSON import (64 static redirects — regex rules added manually per plugin docs).
*   **`PRE-LAUNCH-CHECKLIST.md`**: 11-phase deployment checklist with sign-off table.

## Deployment Instructions

1.  **Setup Fonts (CRITICAL)**:
    *   **Action:** Download the required font files (Roboto Serif & Inter) as detailed in `themes/die-papier-theme/fonts/README.md`.
    *   **Place:** Put the files in `themes/die-papier-theme/fonts/`.
    *   **Reason:** The theme is configured for locally hosted fonts (GDPR/performance) and will display fallbacks if these files are missing.

2.  **Build the Plugin**:
    ```bash
    cd plugins/die-papier-blocks
    npm install
    npm run build
    ```
    Verify all 17 block directories appear in `build/`.

3.  **Install the Theme**:
    *   Compress the `themes/die-papier-theme/` folder into `die-papier-theme.zip`.
    *   Upload to WordPress via **Appearance > Themes > Add New**.

4.  **Install the Plugin**:
    *   Compress the `plugins/die-papier-blocks/` folder (including `build/`) into `die-papier-blocks.zip`.
    *   Upload to WordPress via **Plugins > Add New**.

5.  **Run Content Import**:
    *   From `wordpress-export/`, run: `npm run export:content`.
    *   This generates `die-papier-content.xml`.
    *   **Or use the validation script** (recommended):
        ```bash
        chmod +x scripts/validate-wxr.sh
        ./scripts/validate-wxr.sh
        ```
        This runs the export and then validates the output (XML syntax, content counts, parent hierarchy, nav menus).
    *   Import via **Tools > Import > WordPress**.

6.  **Run Commerce Seed** (optional):
    *   Copy `scripts/seed-commerce.php` to your WordPress root.
    *   Run: `wp eval-file seed-commerce.php` (requires WP-CLI).
    *   This creates the "Digital Subscriber" plan and "Digital Maandeliks" product.

## Dependencies

*   **WordPress 6.9+**: Required for theme.json v3, Interactivity API, and FSE features.
*   **PHP 8.2+**: Required by the plugin.
*   **WooCommerce**: Required for commerce features.
*   **WooCommerce Subscriptions**: Required for recurring payments.
*   **WooCommerce Memberships**: Required for access control.
*   **Secure Custom Fields (SCF) / ACF Pro**: Required for CPT meta fields.
*   **The SEO Framework**: Recommended for SEO.
*   **Node.js 18+**: Required for building blocks and running the content export script.

## Design System

*   **V2 Palette**: Brand red is `#E82C27` (not the older `#D70025`), brand navy is `#172134`.
*   **Typography**: Headings use Roboto Serif (`font-heading`) with `font-normal` weight. Body uses Inter.
*   **Shadows**: Numeric slugs (100–600), not semantic names.
*   **Border Radius**: Uses `--wp--custom--border-radius--*` tokens (not `--wp--preset--border-radius--*`).
*   **Block namespace**: `dp/` prefix in all block.json files.
*   **Pattern slugs**: `die-papier/` prefix for all 70 patterns.
